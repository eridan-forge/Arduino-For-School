# ✨ Урок 4: Умная гирлянда с режимами

**Тип урока:** теория + практика  
**Время:** 45–50 минут  
**Уровень:** начинающие — средний  
**Цель:** собрать гирлянду из 5 светодиодов, управлять режимами одной кнопкой, понять разницу между последовательным и параллельным соединением.

---

## 🧠 Вступление (честный разговор)

> В настоящей ёлочной гирлянде лампочки соединены цепочкой: одна перегорела — всё погасло.  
> Это **последовательное соединение**.

А наша гирлянда будет **параллельной**.  
Каждый светодиод живёт своей жизнью. Если один перестанет работать — остальные продолжают мигать.  
Именно так сделаны уличные гирлянды, светодиодные ленты и иллюминация на домах.

**Почему это круто?**  
Мы можем управлять каждым светодиодом отдельно. У нас будет 5 лампочек и 4 режима работы. Переключать режимы будем одной кнопкой.

---

## 🔌 Схема подключения

| Компонент       | Пин Arduino |
|----------------|-------------|
| Светодиод 1     | D3          |
| Светодиод 2     | D4          |
| Светодиод 3     | D5          |
| Светодиод 4     | D6          |
| Светодиод 5     | D7          |
| Кнопка          | D2 (вторая нога → GND) |

> ⚠️ Каждый светодиод подключается **через резистор 220 Ом** к GND.  
> Кнопка — с внутренней подтяжкой к питанию (как в прошлых уроках).

## 💻 Полный код

```cpp
// пины светодиодов
int leds[] = {3, 4, 5, 6, 7};
int buttonPin = 2;

int mode = 0;        // текущий режим (0-3)
int lastButton = 1;  // для защиты от дребезга

// для режима 0 (звёздное небо)
unsigned long prevMillis[5];
int interval[5];

// для других режимов
unsigned long lastBlink = 0;
unsigned long lastMove = 0;
int currentLed = 0;
bool state = HIGH;

void setup() {
  for (int i = 0; i < 5; i++) {
    pinMode(leds[i], OUTPUT);
    interval[i] = random(200, 2000);
    prevMillis[i] = millis();
  }
  pinMode(buttonPin, INPUT_PULLUP);
}

void loop() {
  // проверка кнопки
  int reading = digitalRead(buttonPin);
  if (reading == LOW && lastButton == HIGH) {
    mode++;
    if (mode > 3) mode = 0;
    delay(200);  // защита от дребезга
  }
  lastButton = reading;

  // выбор режима
  switch (mode) {
    case 0: mode0(); break;
    case 1: mode1(); break;
    case 2: mode2(); break;
    case 3: mode3(); break;
  }
}

// РЕЖИМ 0: ЗВЁЗДНОЕ НЕБО (случайное мигание)
void mode0() {
  for (int i = 0; i < 5; i++) {
    if (millis() - prevMillis[i] >= interval[i]) {
      prevMillis[i] = millis();
      digitalWrite(leds[i], !digitalRead(leds[i]));
      interval[i] = random(200, 2000);
    }
  }
}

// РЕЖИМ 1: ВСЕ ГОРЯТ
void mode1() {
  for (int i = 0; i < 5; i++) {
    digitalWrite(leds[i], HIGH);
  }
}

// РЕЖИМ 2: БЫСТРОЕ МИГАНИЕ (аварийка)
void mode2() {
  if (millis() - lastBlink > 200) {
    lastBlink = millis();
    state = !state;
    for (int i = 0; i < 5; i++) {
      digitalWrite(leds[i], state);
    }
  }
}

// РЕЖИМ 3: БЕГУЩИЙ ОГОНЁК
void mode3() {
  if (millis() - lastMove > 100) {
    lastMove = millis();
    for (int i = 0; i < 5; i++) {
      digitalWrite(leds[i], LOW);
    }
    digitalWrite(leds[currentLed], HIGH);
    currentLed++;
    if (currentLed >= 5) currentLed = 0;
  }
}
🧪 Режимы работы (для детей)
Режим	Название	Что делает
0	Звёздное небо	Каждый светодиод мигает в своём случайном ритме
1	Все горят	Просто постоянный свет
2	Аварийка	Все мигают синхронно каждые 200 мс
3	Бегущий огонёк	Один светодиод бежит по кругу
🎮 Переключение режимов: одна кнопка — нажал и изменил.

🛠️ Задания для учеников
Базовые
Собери схему и загрузи код.

Проверь, что кнопка переключает 4 режима.

Измени скорость бегущего огонька: сейчас 100 мс — сделай 50 или 200.

Продвинутые
Добавь 6-й светодиод (покажи, где в коде нужно добавить пин и расширить логику).

Сделай 5-й режим — «огонёк бежит туда-сюда» (от первого до последнего и обратно).

Эксперт
Добавь потенциометр — пусть он регулирует скорость бегущего огонька в реальном времени.

Сделай так, чтобы при долгом нажатии кнопки режим сбрасывался в 0.

👨‍🏫 Для чего это в кружке
Этот урок даёт ученикам:

понимание параллельного соединения (каждый светодиод независим);

навык программирования нескольких режимов через switch;

уверенную работу с кнопкой (дребезг, состояние);

первые шаги в многозадачности без delay() (режим 0 и 3).

Именно с этой схемы начинаются сложные проекты — светофоры, парктроники, игровые автоматы.
